Requerimientos 
1.	El programa debe registrar los contactos del directorio interno de la empresa
2.	El programa debe buscar los contactos de los empleados en el directorio interno 
3.	El programa debe eliminar los contactos dentro de la empresa 
4.	El Sistema debe permitir al usuario registrar un contacto nuevo  
5.	El Sistema debe permitir al usuario, buscar un contacto por nombre y correo 
6.	El sistema debe registrar al mínimo de un rango de 1 a 2 nombres 
7.	El sistema debe registrar un rango mínimo de 1 a 2 apellidos
8.	El Sistema debe permitir al usuario listar los contactos registrados
9.	El Sistema debe permitir al usuario eliminar un contacto existente 
10.	El programa deber registrar el nombre del contacto
11.	El programa debe registrar el correo electrónico del contacto
12.	El programa debe registrar el cargo del contacto dentro de la empresa 
13.	El programa debe registrar el número de teléfono del contacto 
14.	El nombre y cargo del contacto deben ser de tipo String 
15.	El número del teléfono del contacto debe ser de tipo entero 
16.	El sistema debe permitir al usuario actualizar información del contacto cuando él lo requiera
17.	El sistema debe confirmar mediante una notificación al correo electrónico después de registrarse
18.	El sistema no debe permitir al usuario registrarse con el mismo correo electrónico después de haberse registrado una sola vez 
19.	El sistema debe permitir al usuario enviar un mensaje si no se encuentra un contacto dentro de la lista del directorio 
20.	El sistema debe validar a que no se repita el mismo correo electrónico 
21.	El sistema no puede eliminar un contacto que no se ha registrado o sea inexistente 
22.	El Sistema debe guardar todos los datos de los contactos dentro de un archivo local para ejecutores (contactos.csv)
23.	El sistema debe responder de forma ágil y rápida 
24.	El sistema debe permitir al usuario hacer búsquedas de contactos
25.	El sistema debe mostrar los listados de los contactos 
26.	El sistema puede ser adaptable a cualquier dispositivo móvil 
27.	El sistema debe cumplir las normas de seguridad y privatización de datos
28.	El sistema debe ser intuitivo y de buen uso 
29.	El sistema debe cumplir con una interfaz clara y de mucha facilidad 
30.	El sistema debe cumplir con pistas de menús y mensajes informativos 
31.	El sistema será adaptable para Windows, Mac y Linux 
32.	El sistema debe actualizarse cada 2 o 3 meses 
33.	El sistema incluirá derechos de reservas y copyright
